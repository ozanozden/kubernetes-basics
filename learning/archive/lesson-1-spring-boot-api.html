Lesson 1: Initialize Your Spring Boot API

Goal: Create a simple REST endpoint and initialize git to track your progress.

📚 Knowledge: What You're Building

Before Docker and Kubernetes, you need a working application. You'll create a minimal Spring Boot REST API with a single endpoint that returns a JSON response. This is the foundation you'll containerize and orchestrate later.

Key concepts:

  • @RestController: Spring annotation that marks a class as a web controller returning data (not views)
  • @GetMapping: Maps HTTP GET requests to a handler method
  • ResponseEntity: Represents the entire HTTP response (status code, headers, body)

Read more: Building a RESTful Web Service (Spring.io)

Your Tasks

git init
git add .
git commit -m "Initial Spring Boot project setup"

Why: Git lets me (your teacher) see exactly what you've changed between lessons. Each commit is a checkpoint.

Task 2: Create a REST Controller

Create a new file: src/main/java/com/backend/k8sdemo/HelloController.java

Write a controller with:

  • A @RestController annotation on the class
  • A @GetMapping("/hello") method that returns a ResponseEntity
  • The response should be a simple JSON object like: {"message": "Hello from K8s Demo!", "timestamp": "..."}

Hint: You can use Map.of() to create the JSON response body, and Instant.now() for the timestamp.

✓ Checkpoint: Test Your Endpoint

Before committing, verify your work:

  1. Start the application: ./gradlew bootRun
  2. In another terminal, test the endpoint: curl http://localhost:8080/hello
  3. You should see your JSON response
  4. Stop the app (Ctrl+C)

Task 3: Commit Your Work

Once your endpoint works, commit it:

git add src/main/java/com/backend/k8sdemo/HelloController.java
git commit -m "Add Hello REST endpoint"
