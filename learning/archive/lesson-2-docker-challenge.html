Lesson 2: Containerize with Docker

Goal: Research and containerize your Spring Boot app yourself. No copy-paste allowed! 🚫

⚡ Your Challenge

Package your Spring Boot application into a Docker container and run it.

Success criteria:

  1. Create a Dockerfile that builds your app
  2. Build a Docker image tagged k8s-demo:v1
  3. Run a container from that image
  4. Successfully call curl http://localhost:8080/hello

Key Concepts to Understand

📚 Before You Start

Make sure you understand these concepts as you research:

1. What is a Dockerfile?

A text file with instructions to build a Docker image. Think: recipe for creating a container.

2. Why Multi-Stage Builds?

For Java apps, you need:

  • Build stage: Gradle/Maven + JDK to compile your code
  • Runtime stage: Only JRE + JAR file (smaller, more secure)

3. What Goes in a Dockerfile?

Think about:

  • What base image? (Gradle? JDK? JRE?)
  • How to copy your code into the container?
  • How to build the JAR?
  • How to run the JAR?
  • Which port to expose?

Checkpoints

✓ Checkpoint 1: Dockerfile Created

Create a Dockerfile in your project root.

Questions to answer while writing it:

  • What base image will you use for building? For running?
  • How will you copy dependency files vs source code? (Why does order matter?)
  • What command builds a Spring Boot JAR?
  • Where does Gradle put the built JAR?
  • What command runs a JAR file?

✓ Checkpoint 2: .dockerignore Created

Create a .dockerignore file. Research why this is important!

What to exclude?

  • Build artifacts (.gradle/, build/)
  • Version control (.git/)
  • IDE files (.idea/)
  • Documentation (*.md, learning/)

✓ Checkpoint 3: Build the Image

docker build -t k8s-demo:v1 .

If build fails:

  • Read the error message carefully
  • What layer failed? What was it trying to do?
  • Google the specific error
  • Check file paths, permissions, syntax

Common issues to research:

  • "gradle-wrapper.jar not found docker"
  • "docker copy directory syntax"
  • "spring boot port configuration docker"

✓ Checkpoint 4: Run the Container

docker run -d -p 8080:8080 --name k8s-demo-container k8s-demo:v1

Understand each flag:

  • -d = ?
  • -p 8080:8080 = ?
  • --name = ?

Debug if it doesn't work:

# Check if container is running
docker ps

# View logs
docker logs k8s-demo-container

# Check what port your app listens on
# (Hint: might be in application.yaml!)

✓ Checkpoint 5: Test It!

curl http://localhost:8080/hello

Success: You see your JSON response!

Failure: Read container logs, check port configuration, verify the app started

✓ Checkpoint 6: Commit Your Work

git add Dockerfile .dockerignore
git commit -m "Add Dockerfile for containerization"
