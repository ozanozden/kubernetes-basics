Lesson 2: Containerize with Docker

Goal: Package your Spring Boot app into a Docker container and run it locally.

📚 Knowledge: What is Docker?

Docker packages your application and all its dependencies into a container - a lightweight, standalone executable package. Think of it as a shipping container for your code.

Key Concepts:

  • Image: A blueprint/template for containers (like a class in OOP)
  • Container: A running instance of an image (like an object)
  • Dockerfile: Instructions for building an image
  • Layer: Each Dockerfile instruction creates a layer. Layers are cached for faster builds.

Why Containerize?

  • "Works on my machine" → Works everywhere (same environment)
  • Lightweight: Containers share the OS kernel (unlike VMs)
  • Portable: Run anywhere Docker runs (your laptop, AWS, GCP)
  • Required for Kubernetes: K8s orchestrates containers

Read more: Docker Overview (Docker Docs)

🏗️ Multi-stage Builds

You'll use a multi-stage Dockerfile - a best practice for Java apps. This approach:

  1. Stage 1 (build): Uses Gradle to build your JAR file
  2. Stage 2 (runtime): Copies only the JAR to a minimal JRE image

Why? The final image doesn't include Gradle, source code, or build tools - making it smaller and more secure.

Your Tasks

Task 1: Create a Dockerfile

Create a file named Dockerfile (no extension) in your project root:

/Users/oz.oezden/IdeaProjects/private-projects/k8s-demo/Dockerfile

Your Dockerfile should have two stages:

Stage 1: Build

  • Use gradle:8-jdk21 as the base image
  • Set working directory to /app
  • Copy build files: build.gradle, settings.gradle, gradlew, gradlew.bat, gradle/
  • Copy source code: src/
  • Run the build: ./gradlew bootJar --no-daemon

Stage 2: Runtime

  • Use eclipse-temurin:21-jre as the base image (smaller, only JRE)
  • Set working directory to /app
  • Copy the JAR from stage 1: /app/build/libs/*.jar to /app/app.jar
  • Expose port 8080
  • Set the entrypoint: java -jar /app/app.jar

Hints:

  • Use FROM ... AS build to name the first stage
  • Use COPY --from=build in the second stage to copy from the first
  • Check the Spring Docker guide for syntax examples

Task 2: Create a .dockerignore File

Create .dockerignore in your project root to exclude unnecessary files from the build context:

.git
.gradle
build
.idea
*.md
learning/

Why? This makes builds faster by not sending these files to the Docker daemon.

✓ Checkpoint: Build Your Image

Now test your Dockerfile:

  1. Build the image: 
    docker build -t k8s-demo:v1 .

    This will take a few minutes the first time.

  2. Check the image was created: 
    docker images | grep k8s-demo

    You should see k8s-demo with tag v1

✓ Checkpoint: Run Your Container

Run your containerized app:

  1. Start the container: 
    docker run -d -p 8080:8080 --name k8s-demo-container k8s-demo:v1

    Flags explained:

    • -d: Run in background (detached)
    • -p 8080:8080: Map host port 8080 to container port 8080
    • --name: Name the container for easy reference
  2. Test the endpoint: 
    curl http://localhost:8080/hello

    You should see your JSON response!

  3. View container logs: 
    docker logs k8s-demo-container
  4. Stop and remove the container: 
    docker stop k8s-demo-container
docker rm k8s-demo-container

Task 3: Commit Your Work

Once your container runs successfully:

git add Dockerfile .dockerignore
git commit -m "Add Dockerfile for containerization"

⚠️ Common Mistakes

  • COPY order matters: Copy build.gradle before src/ to leverage layer caching
  • Gradlew permissions: If build fails, make gradlew executable: RUN chmod +x ./gradlew
  • Port mapping: The container port (second number) must match EXPOSE in Dockerfile
